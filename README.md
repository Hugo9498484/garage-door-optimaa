## How it works

1. `.github/workflows/build.yml` runs on every push and rebuilds `public/` from `data/published.json`.
2. `.github/workflows/drip.yml` runs daily at 09:00 UTC, moves `dailyDrip` cities from `data/queue.json` into `data/published.json`, rebuilds, and commits.
3. Vercel deploys `public/` automatically on every commit.

## Deploy to Vercel

1. Import this repo at https://vercel.com/new.
2. **Root Directory:** leave as `.`
3. **Build Command:** `node scripts/build.mjs`
4. **Output Directory:** `public`
5. Deploy.

## Manual controls

- Change drip rate: edit `dailyDrip` in `.seofactory.json`.
- Publish a specific city immediately: move its `{state, city}` entry from `data/queue.json` → `data/published.json` and push.
- Rebuild locally: `node scripts/build.mjs`
